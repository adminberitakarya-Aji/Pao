## 2. How We Use Your Information

### 2.1 Core Service Delivery

- **Conversation Processing**: Your messages are processed by our AI engines to generate companion responses
- **Memory Engine**: Conversations are analyzed to extract and store memories for long-term recall
- **Relationship Engine**: Interactions build your companion's understanding of you (RHI calculation)
- **Proactive Engine**: Patterns trigger timely, relevant outreach from your companion
- **Voice Processing**: Speech-to-text, text-to-speech, timbre verification

### 2.2 Personalization & Improvement

- **Model Training**: Aggregated, anonymized data improves our AI models (opt-in for personal data)
- **Feature Development**: Usage patterns guide new features
- **A/B Testing**: Experiments optimize user experience (opt-out available)
- **Personalization**: Your companion adapts to your communication style, interests, needs

### 2.3 Safety & Security

- **Crisis Detection**: Automated scanning for self-harm, violence, exploitation indicators
- **Intervention**: Graceful resource provision, human escalation when risk is high
- **Fraud Prevention**: Detect abuse, spam, unauthorized access
- **Legal Compliance**: Respond to valid legal requests

### 2.4 Communications

- **Transactional**: Account changes, billing, security alerts (no opt-out)
- **Product Updates**: New features, improvements (opt-out available)
- **Marketing**: Tips, content, promotions (explicit opt-in required)
- **Research**: Optional surveys, interviews (explicit consent each time)

### 2.5 Legal Basis (GDPR/UK)

| Processing Activity | Legal Basis |
|---------------------|-------------|
| Core service (conversations, memory) | Contract performance (Art 6.1.b) |
| Safety/crisis detection | Vital interests (Art 6.1.d) + Legal obligation |

| Analytics, improvement | Legitimate interest (Art 6.1.f) / Consent (Art 6.1.a) |
| Marketing communications | Consent (Art 6.1.a) |

## 4. Data Retention & Deletion

### 4.1 Retention Periods

| Data Category | Retention | Basis |
|---------------|-----------|-------|
| **Account Data** | Active + 90 days after deletion | Contract, legal |
| **Conversation Content** | Active + 90 days after deletion | Contract, user value |
| **Memories** | Active + 90 days after deletion | Contract, user value |
| **Voice Recordings** | 30 days (processing) then deleted | Minimization |
| **Usage/Analytics** | 400 days (13 months) | Legitimate interest |
| **Safety Logs** | 7 years | Legal, vital interests |
| **Payment Data** | 7 years (tax) | Legal obligation |
| **Support Tickets** | 3 years | Legitimate interest |
| **Marketing Consent** | Until withdrawn | Consent |

### 4.2 Deletion Process

1. **User-Initiated**: Settings → Account → Delete Account
2. **Grace Period**: 30 days to recover (soft delete)
3. **Hard Delete**: All personal data purged within 90 days
4. **Backups**: Expire per backup rotation (max 90 days)
5. **Confirmation**: Email confirmation when complete

### 4.3 Data You Can't Delete (Exceptions)

- Anonymized analytics (no personal identifiers)
- Legal hold / litigation preservation
- Fraud prevention records (limited, time-bound)
- Backup remnants (expire automatically)

## 8. Security Measures

### 8.1 Technical Safeguards

- **Encryption**: TLS 1.3 in transit, AES-256 at rest, column-level for sensitive data
- **Authentication**: OAuth 2.0/OIDC, MFA (TOTP, WebAuthn), device fingerprinting
- **Authorization**: Zero-trust, OPA policies, least privilege
- **Network**: mTLS (Istio), network policies, WAF, DDoS protection
- **Containers**: Distroless images, non-root, read-only filesystem, seccomp
- **Secrets**: HashiCorp Vault, dynamic credentials, rotation

### 8.2 Organizational Safeguards

- **Access Control**: Role-based, quarterly reviews, JIT access
- **Training**: Annual security/privacy training, phishing simulations
- **Incident Response**: 24/7 on-call, 15-min SEV-1 response, forensic readiness
- **Audits**: Annual SOC2 Type II, penetration testing, code reviews
- **Vendor Management**: DPA required, security questionnaires, monitoring

### 8.3 Privacy-Specific

- **Data Minimization**: Collect only what's needed
- **Purpose Limitation**: Use only for disclosed purposes
- **Retention Enforcement**: Automated deletion jobs
- **PII Scanning**: Automated detection in logs, code, databases
- **Privacy Impact Assessments**: For new features, vendors, processes

## 9. AI & Automated Processing

### 9.1 How AI Processes Your Data

```
Your Message → Safety Scan → Context Assembly (Memories, RHI, Profile) 
  → LLM Inference → Response Generation → Safety Check → Delivery
       ↓
Memory Extraction → Consolidation → Vector/Graph Storage → Recall Index
```

### 9.2 Model Training

- **Your Data**: NOT used to train base models (OpenAI, Anthropic, etc.)
- **Fine-Tuning**: Only with explicit opt-in (Premium+ feature)
- **Evaluation**: Anonymized samples for quality assessment (opt-out available)
- **Personalization**: In-context learning only (your conversation history)

### 9.3 Transparency

- **Model Cards**: Published for each engine (capabilities, limitations, bias)
- **Evaluation Results**: Quarterly safety/quality reports
- **Decision Logic**: Safety triggers, memory recall criteria documented
- **Human Oversight**: Evaluation engine, safety reviewers, user appeals

## 14. Definitions

| Term | Definition |
|------|------------|
| **Personal Data** | Any information relating to an identified or identifiable natural person |
| **Processing** | Any operation performed on personal data (collection, storage, use, disclosure, deletion) |
| **Controller** | Entity determining purposes/means of processing (PAO) |
| **Processor** | Entity processing on behalf of controller (our vendors) |
| **Data Subject** | You, the individual whose data is processed |
| **PII** | Personally Identifiable Information (US term) |
| **Sensitive Data** | Special category data (health, biometric, sexual orientation, etc.) |
| **Anonymization** | Irreversible removal of identifiability |
| **Pseudonymization** | Reversible replacement of identifiers with tokens |
