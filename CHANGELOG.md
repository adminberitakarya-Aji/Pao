## Versioning Policy

- **Major** (X.0.0): Breaking changes to core engines, architecture, or safety principles
- **Minor** (0.X.0): New capabilities, companion types, or significant features
- **Patch** (0.0.X): Bug fixes, documentation updates, minor improvements

All versions must maintain backward compatibility with user memories and relationships.
